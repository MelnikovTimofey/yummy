# YummyWeb (MVP mobile web)

Mobile-first web клиент для MVP с app-like интерфейсом.

## Запуск

```bash
cd YummyWeb
npm install
npm run dev
```

По умолчанию API: `http://localhost:3001`.

Чтобы переопределить:

```bash
echo "VITE_API_BASE_URL=http://<ваш-host>:3001" > .env.local
```

## UI-стек (актуально)

- `Tailwind CSS` + `shadcn/ui` как базовые UI-примитивы.
- `Radix UI` для headless-поведения (`Dialog`, `Select`, `Tabs`, `ScrollArea`).
- Адаптерный слой `ui-kit` для стабильного продуктового API компонентов.

Ключевые директории:

- `src/components/ui/*` — shadcn-примитивы (низкоуровневый слой).
- `src/ui-kit/*` — продуктовые обёртки (`AppButton`, `AppInput`, `AppSelect`, `AppModal`, `AppBadge`, `AppTabs`).
- `src/ui/*` — экраны и сценарии приложения.
- `src/lib/utils.ts` — утилита `cn(...)` для merge className.

## Правила UI-слоя

1. В экранах (`src/ui/*`) использовать компоненты из `src/ui-kit`, а не прямые HTML-контролы.
2. Если нужен новый базовый примитив — сначала добавить в `src/components/ui`, затем обернуть в `src/ui-kit`.
3. Текущий `src/ui/styles.css` работает в гибридном режиме: legacy-классы + новая токенизированная тема.

## Целевой сценарий после обновления требований

Этот документ фиксирует целевой UX-сценарий и продуктовые требования.  
Он не означает, что все пункты ниже уже полностью реализованы в коде на текущий момент.

- главная состоит из рейлов, где первым всегда идет `Рекомендации для вас`;
- сохраняются текущие типы рейлов (рекомендательные, редакторские, аналитические, мои миксы), добавляется рейл `Избранное` для авторизованного пользователя;
- элемент рейла показывает название микса, названия табаков (без производителя), теги вкусовых профилей, иконку `info` (popup с описанием) и иконку избранного;
- переход в карточку микса выполняется нажатием на элемент (без кнопки `Карточка`);
- нажатие на заголовок рейла или `Смотреть все` открывает отдельный экран полного списка элементов рейла;
- основная tab-навигация: `Главная`, `Каталог`, `Профиль`;
- `Сессии курения` открываются из `Профиля`;
- экран `Подборка` как отдельный сценарий не используется;
- панель `API Endpoint` не является частью продуктового интерфейса.

## Целевая структура экранов

1. Главная с рейлами (`Рекомендации для вас` — первый).
2. Каталог с глобальными фильтрами и поиском по подстроке.
3. Карточка микса (отдельный экран) с переходом по клику на элемент списка/рейла.
4. Полный экран рейла (по `Смотреть все` или клику на заголовок рейла).
5. Профиль с переходами в `Избранное`, `Сессии`, `Предпочтения`, `Создание микса`.

## UI-направление (целевое)

- стилистика: `luxury / lounge / rich / best / old money`;
- CTA `Смотреть все` соответствует общей дизайн-системе продукта;
- согласованные размеры кнопок;
- наличие логотипа;
- для рейлов используется листание кнопками влево/вправо.

## Как войти

1. На экране авторизации введите e-mail и нажмите `Отправить magic link`.
2. Откройте Mailpit (`http://localhost:8025`), скопируйте `token` из ссылки.
3. Вставьте token в поле `Token из письма` и нажмите `Войти`.
