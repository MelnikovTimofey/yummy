# Арома Ателье QA And Smoke

Скилл выбирает наименьший корректный путь проверки, собирает доказательства и
разделяет подтверждённое поведение от неподтверждённого риска.

## Цикл

1. Определить затронутый scope — сопоставить изменение с проектами Арома Ателье.
2. Прогнать baseline build-гейты по каждому изменённому проекту.
3. Выбрать проверку потока: Playwright, когда окружение готово; иначе ручной smoke —
   и явно это указать.
4. Собрать доказательства: результаты команд, скриншоты, URL, viewport, отсутствующие
   предусловия.
5. Отчитаться, разделяя `verified` / `not verified` / `blocked`.

## Baseline build-гейты

- `cd apps/aroma-web && npm run build`
- `cd apps/master-web && npm test && npm run build`
- `cd apps/backend && npm test && npm run build` (Postgres: `npm run db:start`)
- `cd services/telegram-bot && npm test && npm run build`
- UI/интеграция: `cd tests/smoke && npm run smoke`
- `git diff --check`

## UI-проверка

Предпочтительно — целевой Playwright-smoke затронутого потока с фиксацией viewport и
сценария, артефакты при значимом визуальном результате. Fallback — ручной smoke в
браузере: фиксировать URL, viewport, пользовательский путь и явно отметить, что
Playwright не запускался.

## Accessibility-ревью

Когда менялись формы, табы, модалки, фильтры, login или mobile-layout — проверить оси:

1. порядок фокуса и видимый фокус;
2. достижимость основных потоков с клавиатуры;
3. контраст и читаемость русской копии;
4. labels форм, helper-текст, сообщения об ошибках;
5. mobile-читаемость в `Aroma` (поток `18+` → код → CTA, не слишком плотные карточки);
6. role-sensitive поверхности в `Master` (admin-only данные не протекают к `master`).

Каждую находку привязывать к конкретному отказу: недостижимый контрол, неясный label,
нечитаемый контраст, путаница ролей.

## Правила отчёта

- Конкретные находки важнее общих сводок; указывать точные команды и пути проектов.
- Если smoke не запускался — объяснить почему и что осталось непроверенным.
- Не заявлять release-готовность, если выполнены только build-чеки.

## Результат

Затронутый scope; реально выполненные команды; проверенные потоки; находки или явное
«находок нет»; остаточные риски, пробелы покрытия и блокеры окружения.
