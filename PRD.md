# PRD — Web-платформа рекомендаций миксов кальяна (mobile + desktop)

## 1. Обзор продукта

Продукт — web-приложение с пользовательскими сценариями онлайн-кинотеатра:

* главная страница состоит из горизонтальных коллекций (рейлов) миксов;
* пользователь переходит из рейла в карточку микса и выполняет действия;
* персонализация строится на ML-рекомендациях (LightFM) с fallback-логикой.

Фокус MVP — быстро помочь выбрать микс, сохранить его в избранное, оценить и добавить в сессию курения.

---

## 2. Целевая аудитория

* Пользователи, регулярно курящие кальян
* Любители экспериментов со вкусами
* Пользователи, которым нужен быстрый выбор «что покурить сегодня»

---

## 3. Цели продукта

### 3.1 Бизнес / продукт

* Повысить частоту взаимодействий с каталогом и карточками миксов
* Повысить качество персональных рекомендаций
* Увеличить долю сохранений в избранное и добавлений в сессии

### 3.2 Пользовательские

* Быстро находить релевантные миксы
* Сравнивать состав и пропорции в удобной карточке
* Вести историю сессий курения

---

## 4. Режимы и доступ

### 4.1 Режимы интерфейса

* Мобильный режим (mobile-first)
* Десктопный режим (отдельный desktop-сценарий и компоновка)

Оба режима работают в рамках одного web-продукта и одной доменной модели.

### 4.2 Доступ без авторизации

Неавторизованный пользователь может:

* просматривать главную страницу с редакторскими и аналитическими рейлами;
* открывать каталог и карточки миксов;
* пользоваться поиском, фильтрами и сортировкой.

Только в авторизованном режиме доступны:

* персональные рекомендации;
* избранное;
* «мои миксы» и создание миксов;
* добавление микса в сессию;
* оценка миксов;
* профиль и редактирование предпочтений.

---

## 5. Экранная модель (MVP)

### 5.1 Главная

Главная строится из рейлов миксов. Типы рейлов:

1. Рекомендации (персональные / fallback)
2. Редакторские
3. Аналитические
4. Мои миксы

Фиксированный порядок на главной:

1. Рейл рекомендаций (первый и визуально самый крупный)
2. Редакторские рейлы
3. Аналитические рейлы
4. Рейл «Мои миксы»

#### Правила показа рейла рекомендаций

Персональный рейл рекомендаций показывается, если:

* пользователь авторизован;
* и выполнено хотя бы одно условие:
  * пройден онбординг;
  * есть история сессий курения.

Если пользователь авторизован, но условия не выполнены, показывается fallback-рейл рекомендаций.
Если пользователь не авторизован, блок рекомендаций не показывается (на главной остаются редакторские и аналитические рейлы).

#### Редакторские рейлы

* Фиксированный набор в MVP
* Примеры: «Вкусы из детства», «Мужской день», «Вкусы тайги»

#### Аналитические рейлы (обязательные в MVP)

1. ТОП-100 по оценкам
2. ТОП-10 по частоте сессий
3. Новинки
4. Самые добавляемые в избранное

#### Рейл «Мои миксы»

* Доступен только авторизованному пользователю
* Показывает пользовательские миксы текущего пользователя

### 5.2 Каталог

Экран карточек миксов с:

* глобальными фильтрами;
* полнотекстовым поиском;
* сортировками.

Минимально обязательные сортировки:

1. По рейтингу
2. По популярности

### 5.3 Карточка микса (отдельный экран)

Полноценный экран микса с действиями:

1. Добавить в сессию
2. Добавить в избранное
3. Оценить

Обязательные инфографики (круговые диаграммы):

1. По вкусам табака (доли компонентов)
2. По агрегированным вкусовым профилям

### 5.4 Профиль

Основной экран профиля содержит:

1. Имя
2. Переход в избранное
3. Переход в предпочтения
4. Переход в сессии курения
5. Добавить микс

### 5.5 Дополнительные экраны

1. Избранное (список избранных миксов + фильтрация)
2. Предпочтения (редактируемый результат онбординга)
3. Список сессий курения
4. Создание сессии курения
5. Создание микса

---

## 6. Карточка микса в рейле

Элемент любого рейла — карточка микса. Карточка должна содержать:

1. Название микса
2. Состав: вкусы табаков с производителями и пропорциями
3. Визуализацию пропорций (инфографика)

Правила цвета:

* цвета компонентов зависят от вкусовых профилей табаков;
* цвет карточки определяется по вкусовым профилям микса;
* генерация цветов должна минимизировать повторы между карточками в рамках экрана.

## 7. Сущности и доменная модель

### 7.1 Основные сущности

* User
* PreferenceProfile
* Manufacturer
* Tobacco
* Mix
* MixComponent
* SmokingSession
* MixRating
* FavoriteMix

### 7.2 Сущность Mix (обязательные поля)

1. Название
2. Описание
3. Профили вкуса
4. Теги
5. Вкусы табаков с пропорциями

### 7.3 Оценки и сессии

* Оцениваются только миксы
* Шкала оценки микса: 1..5
* Сессии курения не оцениваются

### 7.4 Избранное и мои миксы

* «Избранное» и «Мои миксы» — разные разделы
* Пользователь может добавить в избранное как чужой, так и свой микс

---

## 8. Фильтрация и поиск (глобально)

Глобальные фильтры применимы на любых экранах со списками миксов (главная, каталог, избранное, списки в профиле).

Состав фильтров:

1. Полнотекстовый поиск (название, описание)
2. Профили вкуса (множественный выбор)
3. Теги (множественный выбор)
4. Вкусы табаков
5. Производитель табаков
6. Минимальная оценка (>= N), шкала 1..5

Логика применения:

* между группами фильтров — `AND`;
* внутри группы с множественным выбором — `OR`.

---

## 9. Рекомендательная логика

### 9.1 MVP (v0.1)

Основной подход MVP: ML-модель LightFM.

* Обучение оффлайн
* Предрасчёт рекомендаций и хранение в БД
* Основные сигналы: оценки миксов, история сессий, предпочтения

Приоритет выдачи:

1. Персональная выдача модели
2. Fallback-ранжирование (рейтинг + популярность + диверсификация)

### 9.2 Логика fallback

Fallback обязателен для:

* авторизованных пользователей без достаточных данных;
* случаев недоступности или недостаточной полноты персональной выдачи.

---

## 10. UI/UX-инварианты

1. Интерфейс полностью на русском языке.
2. Mobile-режим учитывает safe area через `env(safe-area-inset-*)`.
3. Mobile-навигация оптимизирована под touch и нижнюю зону управления.
4. Desktop-режим использует отдельную компоновку, адаптированную под широкие экраны (не просто растянутый mobile).
5. На полноэкранных сценариях есть явное действие «Назад»/«Закрыть».
6. Карточки и списки должны быстро сканироваться визуально (четкая иерархия, читабельные подписи, компактные инфографики).

---

## 11. Данные и контент

1. База табаков популярных производителей (минимум: Deus, Darkside, Must Have)
2. База миксов для MVP: 100-200
3. Источники данных — официальные сайты брендов
4. Редакторские коллекции в MVP задаются фиксированным набором

---

## 12. Технический стек и инфраструктура

* Frontend: TypeScript web-клиент
* Backend: Node.js + Fastify + TypeScript
* БД: Postgres + Prisma
* ML: Python + LightFM (offline pipeline)

---

## 13. Нефункциональные требования

* Быстрый отклик API для списков и карточек
* Стабильная пагинация/подгрузка в длинных рейлах (включая ТОП-100)
* Предсказуемая работа фильтров и сортировок

---

## 14. Ограничения

* Контент 18+
* Без социальной ленты
* Без медицинских рекомендаций

---

## 15. Definition of Done (MVP)

Фича считается завершённой, если:

1. Работает end-to-end в mobile и desktop режимах.
2. Соблюдены правила показа рейлов на главной.
3. Глобальные фильтры и сортировки работают консистентно на всех релевантных экранах.
4. Карточка микса содержит действия и обе круговые диаграммы.
5. Оценки применяются только к миксам, шкала 1..5.
6. «Избранное» и «Мои миксы» реализованы как отдельные сценарии.
